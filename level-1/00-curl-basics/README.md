# Level 1 / Phase 00 - curl Basics

## Goal

Prove that an API is just HTTP: a URL, a method, headers, optional body, and a
response.

## Work In

```text
starter/exercises.sh
```

Run each exercise block one at a time instead of running the whole file at once.
Read the command, run it, inspect the output, then move to the next block.

## Key Ideas

- `GET`, `POST`, `PUT`, and `DELETE`
- request and response headers
- status codes
- `curl -v` for debugging
- auth failures with 401 and 403

## Checkpoint

- [ ] What is the difference between `GET` and `POST`?
- [ ] What does `curl -v` show?
- [ ] What is the difference between 401 and 403?
- [ ] Why can the same API be called from curl, Python, or a browser?
